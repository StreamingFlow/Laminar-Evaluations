a workflow to build generates random numbers, keeps only the ones
that are prime, and prints them. It is a short straight-line pipeline of three
stages: a source that produces numbers, a middle stage that tests each number
for primality and lets only primes through, and a sink that prints each prime it
receives.

The first stage is a pure producer with no input of its own, producing a single random integer
between 1 and 1000 and emits it downstream, so over a run it generates a stream of independent
random values, with a random pause between each generation and emission.

The second stage iteratively receives one number at a time and decides whether it is prime.
It also keeps an internal counter of how many numbers it has examined,
incrementing it on each arrival and logging a note that records the number being
checked and the running count. If prime the number is forwarded downstream.

The final stage is a consumer that consumes each prime number it receives and prints
a short message announcing that the number is prime.

The three stages are connected in order — source into the primality tester, and
the primality tester into the printing sink — forming the complete pipeline that
turns a stream of random integers into printed reports of just the prime ones.
