A workflow to build a seismology pipeline that preprocesses ground-motion recordings for a set of stations, applying the
 standard chain of signal-conditioning steps used before cross-correlation studies (of the kind used for ambient-noise
 seismology), and writing the processed result for each station to disk. It is a single long straight-line pipeline: a
 reader loads each station's waveform data, a sequence of signal-processing stages transform it one after another, and
 a final sink saves the result. The unit of data flowing through the pipeline is a small bundle for each station that
 pairs the waveform data (a multi-trace seismic stream) with identifying metadata, and each processing stage modifies
 the waveform in place within that bundle while carrying the metadata along untouched.

The pipeline begins with an iterative reader that is given the path to a text file listing stations. For each station
the reader searches an input directory and reads its seismic stream data, emitting the waveform stream together with
the station name, the counter, and the network code.

From there the waveform passes through a fixed sequence of iterative conditioning stages, each taking the bundle,
operating on the waveform it contains, and passing the bundle onward. First a decimation stage downsamples the waveform
to a target sampling rate. Next a detrend stage removes a simple linear trend from the waveform, and then a demean
stage removes its mean offset. After that a response-removal stage reads the station's instrument-response, returning
only the unit measurement. A bandpass filter stage then restricts the waveform to a chosen frequency band, configured
by minimum and maximum frequencies, a number of filter corners, and whether the filtering is zero-phase.

The waveform then goes through iterative amplitude normalization and spectral whitening. The normalization stage
supports several temporal-normalization methods selected by a parameter. The whitening stage flattens the amplitude
spectrum by dividing the signal's spectrum by a smoothed version of its own amplitude spectrum.

After that an iterative transform stage computes the Fourier transform of the conditioned trace, casting the samples
to a chosen numeric precision, and emitting the transformed data together with the station name and counter.

The final stage is a consumer that writes results to disk, constructing an output directory path built from a fixed
root directory, a data subfolder, a start-time label, and the station name, and then saving the processed data to a
file named from the station and counter. It produces no further output and terminates the pipeline.

The stages are connected in one long chain in exactly this order — reader, decimation, detrend, demean, response
removal, bandpass filter, normalization, whitening, Fourier transform, and finally the file-writing sink — forming
the complete pipeline that turns a list of stations into preprocessed, saved waveform products. Configuration
values are fixed in the assembly: a target of four samples per second for decimation, a pre-filter band and velocity
output units for response removal, default bandpass settings, envelope normalization with a modest window length,
whitening left disabled, and a double-precision FFT with a fixed shift.
